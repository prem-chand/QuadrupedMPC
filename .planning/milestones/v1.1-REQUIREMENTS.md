# Requirements: Milestone v1.1 - MIT Cheetah Parity

**Completed:** 2026-03-27

## Summary

Milestone v1.1 implemented MIT Cheetah parity features:
- State estimation with Kalman filter
- Controller frequency tuning (100 Hz MPC / 500 Hz WBC)
- Balance controller for push recovery

---

## v1 Requirements (Completed)

### Kinematics & Dynamics

- [x] **KND-01**: Forward kinematics computes all 4 foot positions in body frame from joint angles
- [x] **KND-02**: Leg Jacobian matrices computed for all 4 legs (3x3 or 6x3 depending on formulation)
- [x] **KND-03**: Foot velocity computed via Jacobian derivative or numerical differentiation
- [x] **KND-04**: Gravity compensation torques computed correctly for 12-DOF
- [x] **KND-05**: Kinematics verified against MuJoCo built-in functions (误差 < 1e-5)

### IsaacLab Backend

- [x] **ISAAC-01**: IsaacRobot class implements Robot ABC interface
- [x] **ISAAC-02**: get_base_pose() returns position quaternion and RPY
- [x] **ISAAC-03**: get_base_velocity() returns linear and angular velocity
- [x] **ISAAC-04**: get_joint_state() returns joint positions and velocities
- [x] **ISAAC-05**: get_foot_positions_world() returns foot positions in world frame
- [x] **ISAAC-06**: get_leg_jacobian() returns analytical Jacobian for each leg
- [x] **ISAAC-07**: set_torques() applies torques to simulation
- [x] **ISAAC-08**: Single-step controller validation matches MuJoCo behavior

### Batched MPC

- [x] **BATCH-01**: Controller accepts batched state input (num_envs, state_dim)
- [x] **BATCH-02**: WBC computes batched torques (num_envs, 12)
- [x] **BATCH-03**: GPU-accelerated QP solver handles batched problems
- [x] **BATCH-04**: Solver handles infeasible problems gracefully (fallback)
- [x] **BATCH-05**: NaN/Inf detection with CPU fallback

### Robustness

- [x] **ROB-01**: Contact force estimation from residuals
- [x] **ROB-02**: QP solver infeasibility handling with fallback
- [x] **ROB-03**: Frame transformation assertions for debugging
