# Milestone v1.1: MIT Cheetah Parity

**Completed:** 2026-03-27

## Summary

Closed gap between Python prototype and MIT Cheetah controller through:
- State estimation (Kalman filter)
- Controller frequency tuning (100 Hz MPC / 500 Hz WBC)
- Balance controller for push recovery

## Phase Details

### Phase 4: State Estimation (Kalman Filter)

**Goal:** Implement Linear Kalman Filter for robust state estimation matching MIT Cheetah.

**Requirements:** EST-01, EST-02, EST-03

**Success Criteria:**
1. LKF estimates base position/velocity from IMU + encoders
2. Fused orientation from gyro has < 1° drift over 10s
3. Contact-embedded velocity integration reduces position drift

**Artifacts:**
- `go2_mpc/controller/kalman_filter.py` — LinearKalmanFilter, OrientationFilter, ContactEmbeddedVelocity

---

### Phase 5: Controller Frequency & Tuning

**Goal:** Increase control frequencies and tune parameters.

**Requirements:** CTRL-01, CTRL-02, TUNE-01, TUNE-02, TUNE-03

**Success Criteria:**
1. MPC runs at 100 Hz with < 5ms solve time
2. WBC runs at 500 Hz
3. Stable walking on flat ground with tuned parameters

**Artifacts:**
- `go2_mpc/config/config.py` — Updated timing and parameters

**Key Changes:**
- MPC dt: 0.03s → 0.01s (33 Hz → 100 Hz)
- WBC: 100 Hz → 500 Hz
- Q/R weights tuned for Go2 dynamics
- swing_kp: 400 → 500, swing_kd: 10 → 15

---

### Phase 6: Balance Controller

**Goal:** Add push recovery and disturbance rejection.

**Requirements:** BAL-01, BAL-02

**Success Criteria:**
1. Robot recovers from 5N lateral push within 0.5s
2. Reactive gait switching engages correctly on disturbance

**Artifacts:**
- `go2_mpc/controller/balance_controller.py` — BalanceController class

**Key Features:**
- Disturbance detection via roll/pitch/velocity thresholds
- Reactive gait switching to "bound" for lateral disturbances
- Automatic recovery to normal trot after 0.5s
